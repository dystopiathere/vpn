# sing-box based VPN для Windows

Конфигурация [sing-box](https://sing-box.sagernet.org/) в режиме TUN с раздельной маршрутизацией:

- **Браузеры** — российские сайты идут напрямую, всё остальное через прокси
- **Discord** — весь трафик (включая UDP голосовые каналы) через прокси
- **Telegram** — весь трафик через прокси
- **Остальные приложения** (игры и т.д.) — напрямую, прокси их не трогает

Списки приложений и доменов вынесены в отдельные файлы.

---

## Структура проекта

```
vpn/
├── sing-box.exe
├── start.bat                 # ручной запуск (для отладки)
├── service-install.bat       # установка автозапуска (Планировщик задач)
├── service-restart.bat       # перезапуск службы
├── service-uninstall.bat     # удаление автозапуска
├── load-rules.bat            # скачивание geoip rule-set файлов
├── get-outbounds.js          # парсер VLESS-подписки в outbounds
├── confd/
│   ├── 01-main.json          # log, dns, inbounds, route
│   └── 02-outbounds.json     # серверы (генерируется get-outbounds.js)
├── rules/
│   ├── telegram.srs          # IP-диапазоны Telegram
│   └── geoip-ru.srs          # IP-диапазоны России
└── lists/
    ├── apps-browsers.json    # процессы браузеров
    ├── apps-discord.json     # процессы Discord
    ├── apps-telegram.json    # процессы Telegram
    ├── apps-custom.json      # другие процессы
    ├── domains-direct.json   # домены — принудительно напрямую
    └── domains-proxy.json    # домены — принудительно через прокси
```

Все скрипты вычисляют путь до папки проекта автоматически (`%~dp0`) — папку `vpn` можно перенести или переименовать без правок.

## Быстрый старт

1. Скачать или склонировать репозиторий.
2. Получить outbound-серверы из VLESS-подписки:
   ```powershell
   node get-outbounds.js "https://адрес-твоей-подписки"
   ```
3. Скачать geoip rule-set файлы:
   ```powershell
   load-rules.bat
   ```
4. Установить автозапуск (требует прав администратора):
   ```powershell
   service-install.bat
   ```

После этого VPN поднимается автоматически при загрузке Windows.

## Скрипты

### `get-outbounds.js` — генерация серверов из подписки

Парсит VLESS-подписку (base64 список или сырые `vless://` строки) в массив outbound-объектов sing-box, оборачивает их в `urltest`-селектор с автоматическим переключением по пингу.

```powershell
node get-outbounds.js <ссылка-на-подписку-или-файл>
```

Примеры:

```powershell
# по ссылке на подписку
node get-outbounds.js https://example.com/sub/abc123

# из локального файла (например, после ручного экспорта)
node get-outbounds.js sub.txt
```

Запускай заново при смене подписки или добавлении/удалении серверов. **Требует рестарта sing-box.**

### `load-rules.bat` — обновление geoip-списков

Скачивает:

- `telegram.srs` — IP Telegram (runetfreedom/russia-blocked-geoip)
- `geoip-ru.srs` — российские IP-диапазоны (SagerNet/sing-geoip)

```powershell
load-rules.bat
```

Безопасно запускать повторно — файлы перезаписываются. Полезно гонять раз в несколько недель, чтобы списки не устаревали. **Требует рестарта sing-box** после обновления.

### `start.bat` — ручной запуск

Запускает sing-box в текущем окне консоли (с выводом логов), без установки в автозапуск. Удобно для отладки — `Ctrl+C` останавливает процесс. Требует запуска от администратора (TUN-интерфейс).

### `service-install.bat` / `service-restart.bat` / `service-uninstall.bat`

Управляют задачей в Планировщике задач Windows (`AtStartup`, от имени `SYSTEM`, с автоперезапуском при падении). Все три требуют запуска от администратора.

| Скрипт                  | Действие                                                                             |
| ----------------------- | ------------------------------------------------------------------------------------ |
| `service-install.bat`   | создаёт и сразу запускает задачу (если уже существует — ничего не делает)            |
| `service-restart.bat`   | останавливает и заново запускает задачу — применяет правки в `confd/*.json`          |
| `service-uninstall.bat` | останавливает задачу, завершает процесс sing-box.exe, удаляет задачу из планировщика |

Любое изменение `confd/01-main.json` или `confd/02-outbounds.json` требует `service-restart.bat`.

## Настройка кастомных айпи

Файлы `lists/ips-direct.json` и `lists/ips-proxy.json` задают принудительную маршрутизацию по IP-адресам — приоритетнее любых других правил (geoip, процессы и т.д.), действуют для **любого** приложения, не только браузера.

Формат — headless rule-set (`format: "source"`):

```json
{
  "version": 1,
  "rules": [
    {
      "ip_cidr": ["203.0.113.7", "198.51.100.0/24", "198.51.100.7/32", "2001:db8::1"]
    }
  ]
}
```

- **`lists/domains-direct.json`** — сюда добавляй сайты, которые должны открываться напрямую, даже если формально не РФ (например зеркала российских сервисов на зарубежном хостинге).
- **`lists/domains-proxy.json`** — сюда добавляй сайты, которые должны идти через прокси, даже если sing-box посчитал бы их российскими или нерелевантными.

## Настройка кастомных доменов

Файлы `lists/domains-direct.json` и `lists/domains-proxy.json` задают принудительную маршрутизацию по доменам — приоритетнее любых других правил (geoip, процессы и т.д.), действуют для **любого** приложения, не только браузера.

Формат — headless rule-set (`format: "source"`):

```json
{
  "version": 1,
  "rules": [
    {
      "domain_suffix": ["example.com", "another-site.ru"]
    }
  ]
}
```

- **`lists/domains-direct.json`** — сюда добавляй сайты, которые должны открываться напрямую, даже если формально не РФ (например зеркала российских сервисов на зарубежном хостинге).
- **`lists/domains-proxy.json`** — сюда добавляй сайты, которые должны идти через прокси, даже если sing-box посчитал бы их российскими или нерелевантными.

`domain_suffix` матчит домен и все поддомены (`example.com` → `example.com`, `www.example.com`, `api.example.com`). Точное совпадение без поддоменов — поле `domain` вместо `domain_suffix`.

То же самое касается `lists/apps-*.json` — списков процессов:

```json
{
  "version": 1,
  "rules": [
    {
      "process_name": ["chrome.exe", "msedge.exe"]
    }
  ]
}
```

Добавил новый браузер/мессенджер в `apps-browsers.json` — сохранил файл — перезапустил сервис.

## Логика маршрутизации (порядок правил)

| #   | Условие                                                         | Куда   |
| --- | --------------------------------------------------------------- | ------ |
| 1   | приватные IP (локальная сеть)                                   | direct |
| 2   | `lists/domains-direct.json`                                     | direct |
| 3   | `lists/domains-proxy.json`                                      | proxy  |
| 5   | `lists/apps-custom.json`                                        | proxy  |
| 6   | `lists/apps-discord.json`                                       | proxy  |
| 7   | `lists/apps-telegram.json` + IP Telegram                        | proxy  |
| 8   | браузер (`apps-browsers.json`) + российский IP (`geoip-ru.srs`) | direct |
| 9   | браузер, всё остальное                                          | proxy  |
| 10  | всё прочее (игры, прочие приложения)                            | direct |

Правила проверяются по порядку сверху вниз, срабатывает первое совпадение.

## Диагностика

Проверить конфиг без запуска TUN:

```powershell
sing-box.exe check -D . -C confd
```

Посмотреть, какое правило сработало для конкретного соединения — поднять уровень логов в `confd/01-main.json`:

```json
"log": { "level": "debug", "timestamp": true }
```

Запустить вручную и смотреть лог в реальном времени:

```powershell
start.bat
```
